MSME Sentinel — Process Flow (Loan Officer Journey)
From monthly data refresh to underwriter action — how the Early-Warning System is used in practice
01
Monthly Data Refresh
Transactions, GST filings, UPI cash-flow, utility & EPFO records pulled for every active MSME loan account.
→
02
Rank & Trend Update
Model recomputes each account's CMR-style MSME Risk Rank (1–10) and updates its 24-month trajectory.
→
03
RAG Classification
Account bucketed into Green / Amber / Red — mapped onto RBI's SMA-0/SMA-1/SMA-2 — based on rank + deterioration slope.
GREENAMBERRED
→
04
Officer Review (Amber/Red)
Loan officer opens account drill-down: rank, trend chart, plain-English reason codes.
→
05
Human Decision & Action
Officer decides: advisory outreach, restructuring, security review, or monitor. AI informs — never auto-decides.
Feedback loop
06
Outcome Logged & Model Retrained
Actual account outcome (resolved / restructured / defaulted) recorded and used to periodically retrain and recalibrate the scoring model, keeping accuracy current as the portfolio evolves.
