MSME Sentinel — System Architecture
Track 04 · MSME Credit Default Prediction — IDBI Innovate 2026
01 · Data Sources
Core banking transactions & bureau data [Phase 1] · GST returns GSTR-1/3B [Phase 1] · NACH/cheque bounce data [Phase 1] · UPI cash-flow patterns [Phase 2] · EPFO & utility payments [Phase 3, opportunistic]
Phased by real data-infrastructure maturity, not all sourced equally on day one — see docs/data_feasibility_tiering.md
▼
02 · Feature Engineering
Snapshot features (current-month values) + Trend features: 3-month & 6-month rolling slope and volatility per signal — this is what makes early detection possible, not just classification.
▼
03 · Segment-Aware Ensemble Model
XGBoost, class-imbalance corrected (scale_pos_weight). Segment-conditioned on loan type × vintage × borrower category (incl. New-to-Credit handling via alt-data fallback features).
04 · Rank Calibration
Predicted PD → CMR-style 1–10 MSME Risk Rank (fixed decile cutpoints) — matches the real CIBIL MSME Rank convention, not an invented scale. RAG maps directly onto RBI's SMA-0/SMA-1/SMA-2 categories.
▼
05 · Early-Warning Trajectory Engine
Rolling monthly re-score per account. Flags deteriorating slope (not just a point-in-time label) → RAG bucket (Red / Amber / Green) with a 12-month monitoring window.
▼
06 · Explainability Layer
SHAP TreeExplainer → top risk / protective drivers → templated plain-English reason codes for the underwriter.
07 · Loan Officer RAG Dashboard
Portfolio overview + account drill-down: score, trend chart, reason codes, segment metadata.
▼
08 · Human-in-the-Loop Decision
Underwriter reviews score + reasons and takes the credit call. AI advises, does not decide — matching IDBI's explicit requirement.
FEEDBACK LOOP Actual outcomes (recovered / restructured / defaulted) flow back into periodic model retraining to keep the score calibrated as the portfolio evolves.
